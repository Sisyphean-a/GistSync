# GistSync

GistSync 是一个基于 Wails（Go + Vue）的桌面工具，用于将本地配置文件加密后同步到 GitHub Gist，并在另一台电脑按需恢复。

## 项目作用

- 解决多台电脑之间配置文件同步问题（如 `.gitconfig`、`ssh` 相关配置等）。
- 上传前使用主密码进行加密，避免明文直接存储在云端。
- 以“配置集（Profile）+ 快照（Snapshot）”形式管理同步历史。

## 核心能力

- 多配置集管理：创建、切换、删除配置集。
- 条目级文件管理：可为配置集添加多个文件路径。
- 条目级选择性同步：
  - 上传时可只上传选中的文件条目。
  - 恢复时可只恢复选中的文件条目。
- 冲突预检与显式决策：
  - 目标文件已存在时，默认跳过。
  - 支持逐项覆盖，或本次全选覆盖/全选跳过。

## 技术栈

- 后端：Go 1.23
- 桌面框架：Wails v2
- 前端：Vue 3 + TypeScript + TailwindCSS
- 云端存储：GitHub Gist API

## 开发与运行

### 1. 环境准备

- Go 1.23+
- Node.js 18+
- Wails CLI（已安装并可执行 `wails`）

### 2. 安装依赖

```bash
cd frontend
npm install
```

### 3. 开发模式

```bash
cd ..
wails dev
```

### 4. 构建可执行文件

```bash
wails build
```

构建产物默认位于：

`build/bin/GistSync.exe`

## 使用流程（当前界面）

应用分为三个主区：`同步中心`、`配置管理`、`安全设置`。

### 1) 安全设置

1. 进入 `安全设置`。
2. 填写 GitHub Token 与主密码（Master Password）。
3. 点击保存。

### 2) 配置管理

1. 创建一个配置集。
2. 在该配置集下添加要同步的本地文件（支持多选）。
3. 如需调整恢复策略，可设置：
   - 原始路径恢复（original）
   - 指定根目录恢复（rooted）

### 3) 同步中心

上传：

1. 选择配置集。
2. 勾选要上传的条目。
3. 点击 `上传选中条目`。

恢复：

1. 选择配置集与快照。
2. 勾选要恢复的条目。
3. 点击 `预检冲突并应用快照`。
4. 若检测到冲突，在冲突面板中逐项决定覆盖或跳过（默认跳过）。

## 安全说明

- Token 与主密码保存在本地设置文件中。
- 云端文件内容为加密后文本，不存储明文内容。
- 主密码错误会导致恢复解密失败，这是预期安全行为。

## 常见问题

### 1. 启动后窗口空白

通常是前端资源未正确加载或运行时异常导致。建议按以下顺序确认：

1. 先执行 `npm run build`（frontend）确保前端可构建。
2. 执行 `wails build` 重新打包。
3. 运行最新产物 `build/bin/GistSync.exe`。

### 2. 恢复时为什么没覆盖本地文件

这是默认安全策略：冲突项默认跳过。需要在冲突决策面板里显式勾选覆盖后才会写入。
