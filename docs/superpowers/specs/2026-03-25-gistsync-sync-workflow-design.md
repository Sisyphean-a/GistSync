# GistSync 双机选择性同步与界面重构设计（2026-03-25）

## 1. 背景与目标
当前版本已具备加密同步、配置集、快照能力，但前端交互混杂，下载冲突通过 `window.prompt` 完成，容易误操作。

本次重构目标：
- 优先解决“防误覆盖”问题。
- 支持双机场景下按文件选择性同步。
- 在关键流程可用性提升后，统一视觉层级与信息结构。

## 2. 用户场景与优先级
用户核心场景：
- 两台电脑之间同步配置。
- 在目标机器按需选择部分配置文件恢复。

风险优先级：
1. 误覆盖本地文件（最高）。
2. 步骤过多导致误操作。
3. 不清楚同步结果。

冲突策略：
- 默认逐项确认，并提供“本次全选覆盖/全选跳过”。
- 默认状态为“跳过”，只有显式选择覆盖才会写入。

## 3. 信息架构与交互
顶层改为三大区：
- 同步中心
- 配置管理
- 安全设置

### 3.1 同步中心
上传流程：
1. 选择配置集。
2. 勾选要上传的文件条目。
3. 执行上传并返回快照与统计。

恢复流程：
1. 选择配置集与快照。
2. 可选勾选要恢复的文件条目。
3. 预检冲突并进入冲突决策面板。
4. 明确覆盖/跳过后执行应用。
5. 展示结果清单（成功/跳过/错误）。

### 3.2 配置管理
- 新建、切换、删除配置集。
- 添加文件条目（多选）、移除条目。
- 配置恢复策略（原路径/根目录）。

### 3.3 安全设置
- Token、主密码独立管理。
- 错误信息显式暴露，不做静默降级。

## 4. 技术设计

### 4.1 前端拆分
将当前 `SyncPanel.vue` 按职责拆分：
- `SyncCenter.vue`：上传/恢复流程编排。
- `ProfileManager.vue`：配置与条目管理。
- `ConflictResolverDialog.vue`：冲突决策。
- `SnapshotPicker.vue`：快照选择。

状态分层：
- 持久状态：设置与配置集（后端）。
- 会话状态：本次选择项、冲突决策。
- 派生状态：可执行条件、统计摘要。

### 4.2 后端接口改造
`ApplySnapshotRequest` 新增：
- `SelectedItemIDs []string`：仅应用这些条目，空表示全部。

`UploadProfileRequest` 新增：
- `SelectedItemIDs []string`：仅上传这些条目，空表示全部。

规则：
- 非法参数直接返回错误。
- 不新增 silent fallback。

### 4.3 冲突处理机制
- 冲突检测保持 `PreviewApplyConflicts`。
- `ApplySnapshot` 仅在 `OverwriteItemIDs` 中的冲突项允许覆盖。
- 未显式授权覆盖则跳过并返回结果明细。

## 5. 视觉规范
- 主操作、危险操作、次操作保持固定语义色。
- 左侧导航 + 右侧工作区，减少信息噪音。
- 统一按钮、间距、卡片、表格样式 token。
- 不依赖 prompt 弹窗，统一应用内面板。

## 6. 验收标准
1. A 机上传后，B 机可只恢复部分文件。
2. 冲突默认跳过，显式覆盖才写入。
3. 执行前后均可查看覆盖/跳过/失败统计。
4. 同步流程每一步有明确反馈。

## 7. 实施边界
本轮包含：流程重构、冲突决策 UI、选择性同步、视觉统一。
本轮不包含：复杂文件 diff 编辑器、历史审计中心、自动三方合并。
